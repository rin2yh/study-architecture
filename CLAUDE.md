# 基本方針

- 敬語で話す
- 推測するな、計測せよ
- 設計判断はADRで記載する
- 怪しい箇所や判断を仰ぐべき箇所はask toolを使用する

# コメント方針

- **what コメント禁止**: コードを読めば分かること（処理内容の言い換え）はコメントに書かない。
  コメントは **why（なぜそうするか・背景・非自明な制約）** だけを書く。
- 対象は **全言語**（Go / TypeScript / SQL / YAML / shell など、go 以外も含む）。
- 例:
  - ✗ `// rows を id 昇順で詰め直す` （直後のコードそのまま = what）
  - ✓ `// pgxpool.New は遅延接続なので、不正な DSN でもここでは error にならない` （why）
- 関数名・変数名・型で意図が伝わるなら、説明コメントは足さない（命名で語らせる）。
- 例外: 公開 API のドキュメントとして必要な doc コメント、生成コードの先頭マーカー等は残してよい。
