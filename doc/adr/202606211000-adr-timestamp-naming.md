# ADR-202606211000: ADR の識別子を連番からタイムスタンプ (YYYYMMDDHHmm) に変える

- Status: Accepted
- Date: 2026-06-21
- Supersedes: これまでの連番採番 (`NNNN-slug.md`) 規約
- Relates to: `.claude/rules/comments.md` (ADR 参照の書式), `README.md` (ADR リンク)

## Context

ADR を `0001-slug.md` のような **4 桁連番**で採番していた。次番号を採るには既存の最大番号を
知る必要があり、ブランチを分けて並行に ADR を起こすと**両方が同じ次番号を選ぶ**。Git は
別ファイル名どうしを衝突として検出しないので、マージは通ってしまい**重複番号が黙って残る**。

実際にこのリポジトリでも `0014` (api-error-model / repository-real-db-integration-test) と
`0016` (client-generated-code-layout / parallel-integration-test-template-db) が重複していた。
重複は `[[NNNN]]` 参照の宛先を曖昧にする (どちらの 0014 か文脈で判断するしかない)。

採番の調停 (誰がどの番号を採るか) を**作成時点で不要**にしたい。

## Decision

**ADR の識別子をファイル名先頭の作成日時タイムスタンプ `YYYYMMDDHHmm` にする。**
ファイル名は `YYYYMMDDHHmm-slug.md`、相互参照は `ADR-[[YYYYMMDDHHmm]]` 形式。

- **粒度は分 (`YYYYMMDDHHmm`)**。日付のみだと同日 2 件目でまた衝突する。分まで入れれば
  人手で同一分に 2 件起こす状況はまず無く、起きても作成者が片方を 1 分ずらせば済む。秒まで
  入れると識別子が冗長になる割に衝突確率は実用上変わらないため、分で止める。
- **見出しは `# ADR-<timestamp>: <title>`**。本文 1 行目に維持する。
- **相互参照は `ADR-[[<timestamp>]]`**。`ADR-` 接頭辞を付け、素の `[[…]]` より参照と分かる
  ようにする (12 桁の数字列だけだと何の ID か読み取れないため)。
- **既存 18 件もすべて改名して統一**する。識別子は各 ADR の `Date:` を日付に採り、同日内は
  元の連番順に分を割り当てて一意化した (例: 2026-06-17 の群は `09:00` から 1 分刻み)。重複
  していた `0014`/`0016` もこれで別 ID になる。
- 参照 (`[[NNNN]]` → `ADR-[[…]]`) は ADR 本文・コード中コメント (`checkout.go` /
  `*.sql`)・README リンク・`comments.md` の例とフックのプロンプトまで一括で追従させる。

## Consequences

- **作成時の番号調停が消える**。新規 ADR は「今の時刻 + slug」で名前が決まり、並行作成でも
  分が違えば自然に一意。連番由来の衝突 (本 ADR の動機) が原理的に起きなくなる。
- **識別子から作成順が読める**。連番でも順序は分かったが、連番は採番のタイミングで前後し得た
  (例: 旧 `0008` は日付上は 06-19 で `0010`〜`0012` より後)。タイムスタンプは作成日時そのもの。
- **参照が長くなる**。`[[0008]]` → `ADR-[[202606190900]]`。可読性は接頭辞で補う。slug や短縮 ID を
  併用する案は不採用 (下記)。
- **過去 ID への外部リンクが切れる**。学習用リポジトリで外部参照は無く、影響は許容。今後は
  改名を避け、Superseded でも**ファイル名は据え置き**にする。
- 既存の `[[NNNN]]` を機械置換したため、**新規 ADR は必ず `ADR-[[…]]` で書く** (フックのプロンプトも
  この書式を ADR 参照として除外するよう更新済み)。

## Alternatives considered

- **連番のまま運用ルールで衝突回避**: 「PR 前に番号を採り直す」等。人間の規律に頼るため、
  まさにそれが破れて重複が出ている。仕組みで消すべき。
- **短い安定 ID を別に持つ (`ADR-19` 等) + 日付プレフィックスのファイル名**: 参照は短く保てるが、
  その短 ID の採番でまた調停が要る (= 連番と同じ衝突源)。本 ADR の目的に反する。
- **slug で参照する (`ADR-[[cross-domain-snapshot]]`)**: 人に読みやすいが、slug の改名で参照が
  全部切れる。タイトル変更が ADR 更新の常で、参照を脆くするため不採用。
- **秒粒度 (`YYYYMMDDHHmmss`)**: 衝突は分粒度でほぼ消えており、秒は識別子を伸ばすだけで
  対価が薄い。分で止める。
