# テスト規約

新規追加するテストは原則テーブル駆動で書く。

- **Go**: `tests := []struct{ name string; ... }{...}` を作り `for _, tt := range tests { t.Run(tt.name, func(t *testing.T) { ... }) }` で回す
- **TS (vitest)**: 同じ shape を持つケースなら `it.each([...])` を使う

理由: 同種のケースを一覧で見せた方が抜けが分かりやすく、ケース追加も「行を増やすだけ」になる。

ただし次の場合はテーブル化を**しない**:

- テーブル化のために本体に `if` / `switch` や追加ヘルパー関数を増やすことになる
- ケースが 1 つだけ、あるいは shape が大きく違う異質ケースだけ
- 既存の素直な `Test...` / `it(...)` を **テーブル化のためだけ** に書き直す (新規追加でテーブルにする方が安い)

可読性とテーブル化の天秤で、テーブル化が「行が増えるだけ」で済まないなら個別テストで書く。
